## 基础问题

### Q1: 什么是端到端加密密码分享？

**A:** 端到端加密意味着你的密码在发送前就已经被加密，只有接收方才能解密。即使通过我们的服务器传输，服务器也无法看到明文密码。这确保了只有你和接收方能访问分享的内容。

### Q2: 为什么需要添加联系人？

**A:** 添加联系人的过程本质上是交换公钥的过程。通过扫描对方的身份码，你获得了他们的公钥，这样你就可以加密密码给他们。同样，他们也获得你的公钥，可以给你分享密码。

### Q3: 接收方必须安装 SafeVault 吗？

**A:** 是的。由于我们使用端到端加密，接收方必须安装 SafeVault 应用，并且拥有对应的私钥才能解密分享的密码。

### Q4: 可以给没有 SafeVault 的人分享吗？

**A:** 目前不可以。接收方必须先安装 SafeVault 并注册。不过，你可以邀请他们安装 SafeVault，然后通过扫描身份码的方式添加为联系人。

## 安全问题

### Q5: 分享的密码会被服务器看到吗？

**A:** 不会。SafeVault 采用零知识架构：
- 发送方使用接收方的公钥加密
- 服务器只存储加密后的数据
- 只有接收方的私钥可以解密
- 服务器无法获取明文密码

### Q6: 如果我的手机被盗，别人能使用我的身份吗？

**A:** 不能。要使用分享功能，需要：
1. 解锁主密码库（需要主密码或生物识别）
2. 使用私钥解密分享（需要主密码派生的私钥）

只要主密码安全，身份就是安全的。

### Q7: 联系人的公钥存储在哪里？

**A:** 联系人的公钥加密存储在你的本地设备上：
- 不会上传到服务器
- 使用你主密码派生的密钥加密
- 只有你能访问这些公钥

### Q8: 如果我换了手机，我的联系人还在吗？

**A:** 这取决于你的同步设置：
- 如果你启用了云端同步，联系人会同步到新设备
- 如果只使用本地存储，需要重新添加联系人
- 建议定期备份联系人数据

### Q9: 可以撤销已发送的分享吗？

**A:** 可以。在「分享历史」中：
1. 找到要撤销的分享
2. 点击「撤销」
3. 已接收的分享仍然有效
4. 但未接收的分享将失效

## 使用问题

### Q10: 如何分享我的身份码？

**A:** 有以下几种方式：
1. **当面扫描**：打开「我的身份码」页面，让对方扫描
2. **图片分享**：将身份码截图发送（微信、QQ等）
3. **复制文本**：复制身份码文本内容发送

### Q11: 分享时提示"数据过大"怎么办？

**A:** RSA加密有数据大小限制（约245字节）：
- 尝试减少密码条目的备注信息
- 使用「离线文件分享」功能
- 或者只分享核心密码信息

### Q12: 接收分享后如何保存？

**A:** 接收分享后：
1. 查看分享的密码详情
2. 点击「保存到密码库」
3. 可以选择保存位置和分类
4. 保存后可以在密码库中找到

### Q13: 分享的有效期是什么意思？

**A:** 有效期控制分享链接的存活时间：
- **1小时**：适合临时查看（如维修人员需要WiFi密码）
- **1天**：适合短期分享（如借给朋友使用）
- **7天**：适合一般分享（如家庭共享）
- **永不过期**：适合常用联系人（谨慎使用）

### Q14: 可以修改已发送分享的权限吗？

**A:** 目前不可以。如果需要修改权限：
1. 撤销原分享
2. 重新创建分享并设置新权限

## 技术问题

### Q15: 使用什么加密算法？

**A:** SafeVault 使用行业标准的加密算法：
- **密钥派生**: PBKDF2-HMAC-SHA256 (100,000迭代)
- **密钥类型**: RSA-2048
- **加密模式**: RSA-OAEP with SHA-256
- **签名算法**: SHA256withRSA

### Q16: 为什么密钥对是从主密码派生的？

**A:** 这样设计有以下好处：
- 不需要单独备份密钥
- 换设备时恢复主密码即可恢复密钥
- 减少用户管理负担
- 保持密钥一致性

### Q17: 身份码包含什么信息？

**A:** 身份码是一个 JSON 对象，包含：
```json
{
  "username": "用户名",
  "publicKey": "PEM格式的公钥",
  "keyId": "密钥ID",
  "timestamp": "生成时间戳",
  "signature": "对上述内容的签名"
}
```

### Q18: 分享码包含什么信息？

**A:** 分享码包含加密后的密码数据：
```json
{
  "senderId": "发送方ID",
  "receiverId": "接收方ID",
  "encryptedData": "Base64编码的加密数据",
  "signature": "发送方签名",
  "expiresAt": "过期时间戳"
}
```

### Q19: 离线文件格式是什么？

**A:** 离线文件使用 .svsf 扩展名（SafeVault Share File）：
- 包含加密的密码数据
- 可以通过任何方式传输（U盘、邮件、聊天应用）
- 接收方用 SafeVault 打开即可

### Q20: 如何验证联系人的身份？

**A:** 建议采取以下措施：
- **当面交换**：最好面对面扫描身份码
- **视频确认**：通过视频通话确认对方身份
- **二次验证**：通过其他渠道验证（如电话）
- **检查备注**：给联系人添加清晰的备注

## 故障排除

### Q21: 扫描身份码失败怎么办？

**A:** 尝试以下方法：
1. 确保二维码清晰可见
2. 调整摄像头距离和角度
3. 确保光线充足
4. 尝试手动输入身份码

### Q22: 分享后对方无法接收怎么办？

**A:** 检查以下几点：
1. 确认对方已安装 SafeVault
2. 确认对方已完成注册
3. 检查网络连接
4. 确认分享未过期
5. 让对方刷新分享列表

### Q23: 提示"签名验证失败"是什么意思？

**A:** 这通常表示：
- 身份码可能被篡改
- 确认身份码的来源
- 重新从可信渠道获取身份码
- 联系对方重新生成

### Q24: 解密分享时提示错误？

**A:** 可能的原因：
1. 主密码错误（无法派生私钥）
2. 分享数据损坏
3. 分享已过期或被撤销
4. 联系人公钥不匹配

### Q25: 应用崩溃或数据丢失？

**A:** 立即采取以下措施：
1. 重启应用
2. 检查网络连接
3. 如果数据丢失，从备份恢复
4. 联系技术支持：support@safevault.app

### Q26: 如何备份我的联系人？

**A:** 目前有以下方式：
1. 导出联系人列表（功能开发中）
2. 截图保存身份码
3. 使用手机备份功能
4. 启用云端同步（推荐）

## 隐私问题

### Q27: SafeVault 会收集我的位置信息吗？

**A:** 不会。SafeVault 的分享功能：
- 不使用位置服务
- 不收集位置数据
- 不需要任何位置权限

### Q28: 联系人关系会上传到服务器吗？

**A:** 不会。联系人关系：
- 只存储在你的本地设备
- 不会同步到服务器
- 不会被用于社交网络分析

### Q29: 分享记录会保留多久？

**A:** 分享记录：
- 永久保存在本地（除非手动删除）
- 加密存储
- 可以在「分享历史」中查看和管理
- 过期的分享会标记为失效

### Q30: 我的数据会被用于广告吗？

**A:** 绝对不会。SafeVault 承诺：
- 不出售用户数据
- 不使用数据做广告定向
- 不与第三方共享数据
- 遵守隐私法规

## 其他问题

### Q31: 可以为一个密码创建多个分享吗？

**A:** 可以。你可以：
- 为同一个密码创建多个分享
- 每个分享可以有不同接收方
- 每个分享可以设置不同权限和过期时间

### Q32: 可以批量分享密码吗？

**A:** 目前不支持批量分享。每个密码需要单独分享。未来版本可能会添加此功能。

### Q33: 可以分享密码夹吗？

**A:** 目前不可以。只能分享单个密码条目。密码夹分享功能在规划中。
