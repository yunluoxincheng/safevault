## 基础问题

### Q1: 什么是端到端加密密码分享？

**A:** 端到端加密意味着你的密码在发送前就已经被加密，只有接收方才能解密。即使通过我们的服务器传输，服务器也无法看到明文密码。这确保了只有你和接收方能访问分享的内容。

### Q2: 为什么必须先添加好友才能分享？

**A:** 为了确保分享的安全性和可控性，SafeVault v2.2 要求只能与已添加的好友分享密码。这样可以：
- 防止陌生人随意请求分享
- 确保你认识分享对象
- 提高分享的可控性
- 减少安全风险

### Q3: 接收方必须安装 SafeVault 吗？

**A:** 是的。由于我们使用端到端加密，接收方必须安装 SafeVault 应用，并且拥有对应的私钥才能解密分享的密码。

### Q4: 可以给不是好友的人分享吗？

**A:** 不可以。从 v2.2 版本开始，SafeVault 只支持与已添加的好友分享密码。如果需要分享给某人，请先添加对方为好友。

## 安全问题

### Q5: 分享的密码会被服务器看到吗？

**A:** 不会。SafeVault 采用零知识架构：
- 发送方使用接收方的公钥加密
- 服务器只存储加密后的数据
- 只有接收方的私钥可以解密
- 服务器无法获取明文密码

### Q6: 如果我的手机被盗，别人能使用我的身份吗？

**A:** 不能。要使用分享功能，需要：
1. 解锁主密码库（需要主密码或生物识别）
2. 使用私钥解密分享（需要主密码派生的私钥）

只要主密码安全，身份就是安全的。

### Q7: 好友关系存储在哪里？

**A:** 好友关系存储在云端服务器：
- 用于在不同设备间同步
- 可以在「好友」中查看和管理
- 删除好友不会影响已创建的分享

### Q8: 如果我换了手机，我的好友还在吗？

**A:** 是的。好友关系存储在云端，换设备后登录账号即可同步所有好友关系。

### Q9: 可以撤销已发送的分享吗？

**A:** 可以。在「分享」→「已发送」中：
1. 找到要撤销的分享
2. 点击「撤销」
3. 分享状态变为「已撤销」
4. 接收方将无法访问该分享

## 使用问题

### Q10: 如何添加好友？

**A:** 有以下几种方式：
1. **搜索用户名**：输入对方的用户名搜索并发送请求
2. **扫描二维码**：扫描对方的身份码自动发送请求
3. **输入分享码**：粘贴对方的身份码内容添加

### Q11: 好友请求会过期吗？

**A:** 好友请求不会自动过期，但建议定期清理未处理的请求。

### Q12: 如何删除好友？

**A:** 在「好友」列表中：
1. 点击好友旁边的「更多」
2. 选择「删除好友」
3. 确认删除

删除后不能再与该好友分享密码。

### Q13: 删除好友后，之前的分享还有效吗？

**A:** 是的。删除好友关系不会影响已创建的分享。已发送的分享仍然有效，直到过期或被撤销。

### Q14: 接收分享后如何保存？

**A:** 接收分享后：
1. 查看分享的密码详情
2. 点击「保存到密码库」
3. 可以选择保存位置和分类
4. 保存后可以在密码库中找到

### Q15: 分享的有效期是什么意思？

**A:** 有效期控制分享的存活时间：
- **1小时**：适合临时查看
- **1天**：适合短期分享
- **7天**：适合一般分享
- **永不过期**：适合常用好友（谨慎使用）

### Q16: 可以修改已发送分享的权限吗？

**A:** 目前不可以。如果需要修改权限：
1. 撤销原分享
2. 重新创建分享并设置新权限

## 技术问题

### Q17: 使用什么加密算法？

**A:** SafeVault 使用行业标准的加密算法：
- **密钥派生**: PBKDF2-HMAC-SHA256 (100,000迭代)
- **密钥类型**: RSA-2048
- **加密模式**: RSA-OAEP with SHA-256
- **签名算法**: SHA256withRSA

### Q18: 为什么密钥对是从主密码派生的？

**A:** 这样设计有以下好处：
- 不需要单独备份密钥
- 换设备时恢复主密码即可恢复密钥
- 减少用户管理负担
- 保持密钥一致性

### Q19: 身份码包含什么信息？

**A:** 身份码是一个 JSON 对象，包含：
```json
{
  "username": "用户名",
  "publicKey": "PEM格式的公钥",
  "keyId": "密钥ID",
  "timestamp": "生成时间戳",
  "signature": "对上述内容的签名"
}
```

### Q20: 分享数据如何存储？

**A:** 分享数据存储在云端服务器：
- 使用接收方的公钥加密
- 服务器无法解密
- 只有接收方可以解密
- 支持多设备同步

### Q21: 如何验证好友的身份？

**A:** 建议采取以下措施：
- **当面交换**：最好面对面扫描身份码
- **视频确认**：通过视频通话确认对方身份
- **二次验证**：通过其他渠道验证（如电话）
- **检查备注**：给好友添加清晰的备注

## 故障排除

### Q22: 扫描身份码失败怎么办？

**A:** 尝试以下方法：
1. 确保二维码清晰可见
2. 调整摄像头距离和角度
3. 确保光线充足
4. 尝试手动输入身份码

### Q23: 分享后对方无法收到怎么办？

**A:** 检查以下几点：
1. 确认对方已安装 SafeVault v2.2+
2. 确认对方已完成注册
3. 检查网络连接
4. 确认双方是好友关系
5. 让对方刷新分享列表

### Q24: 提示"不是好友"怎么办？

**A:** 这表示：
1. 对方不在你的好友列表中
2. 请先添加对方为好友
3. 等待好友请求被接受
4. 然后再尝试分享

### Q25: 解密分享时提示错误？

**A:** 可能的原因：
1. 主密码错误（无法派生私钥）
2. 分享数据损坏
3. 分享已过期或被撤销
4. 好友关系已变更

### Q26: 应用崩溃或数据丢失？

**A:** 立即采取以下措施：
1. 重启应用
2. 检查网络连接
3. 如果数据丢失，从备份恢复
4. 联系技术支持：support@safevault.app

### Q27: 如何备份我的好友关系？

**A:** 好友关系存储在云端：
1. 登录账号即可同步
2. 不需要手动备份
3. 可以在「好友」中查看和管理

## 隐私问题

### Q28: SafeVault 会收集我的位置信息吗？

**A:** 不会。SafeVault v2.2 移除了所有位置相关功能：
- 不使用位置服务
- 不收集位置数据
- 不需要任何位置权限

### Q29: 好友关系会被用于社交网络分析吗？

**A:** 不会。SafeVault 承诺：
- 好友关系仅用于密码分享
- 不会用于社交推荐
- 不会用于广告定向
- 不会与第三方共享

### Q30: 分享记录会保留多久？

**A:** 分享记录：
- 永久保存在云端
- 加密存储
- 可以在「分享」中查看和管理
- 过期的分享会标记为「已过期」

### Q31: 我的数据会被用于广告吗？

**A:** 绝对不会。SafeVault 承诺：
- 不出售用户数据
- 不使用数据做广告定向
- 不与第三方共享数据
- 遵守隐私法规

## 其他问题

### Q32: 可以为一个密码创建多个分享吗？

**A:** 可以。你可以：
- 为同一个密码创建多个分享
- 每个分享可以发给不同好友
- 每个分享可以设置不同权限和过期时间

### Q33: 可以批量分享密码吗？

**A:** 目前不支持批量分享。每个密码需要单独分享。未来版本可能会添加此功能。

### Q34: 可以分享密码夹吗？

**A:** 目前不可以。只能分享单个密码条目。密码夹分享功能在规划中。
