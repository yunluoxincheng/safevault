### 7. 详细说明页面

**标题**: 关于新加密算法

**简介**:
```
SafeVault v3.6.0 引入了现代椭圆曲线加密算法 X25519 和 Ed25519，
这是业界领先的安全标准。
```

**X25519 密钥交换**:
```
用于安全地交换密钥，支持前向保密。每次分享使用新的临时密钥，
确保即使长期私钥泄露，历史分享仍安全。
```

**Ed25519 数字签名**:
```
用于验证发送方身份。比传统 RSA 签名快 100 倍，同时提供更强的安全性。
```

**HKDF 密钥派生**:
```
从 ECDH 共享密钥派生 AES 密钥，同时混合双方身份，防止密钥混淆攻击。
```

### 10. 常见问题页面

**标题**: 常见问题

**Q1: 迁移会删除我的密码吗？**
A: 不会。迁移只是生成新的加密密钥对，所有密码数据保持不变。

**Q2: 迁移需要多长时间？**
A: 通常 < 1 秒。迁移只是生成密钥并上传，不需要加密/解密现有数据。

**Q3: 迁移失败怎么办？**
A: 迁移是幂等的，失败后可以重试。旧密钥不会被删除，可以安全回滚。

**Q4: 必须迁移吗？**
A: 不是必须的。旧版本将继续正常工作，但无法享受新算法的性能优势。

**Q5: 迁移后还能和旧版本用户分享吗？**
A: 可以。系统会自动检测对方的加密算法版本，使用双方都支持的协议。
