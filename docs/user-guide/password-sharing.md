# SafeVault 密码分享功能用户指南

## 功能概述

SafeVault 支持**端到端加密**的密码分享功能，确保你的密码安全地分享给家人、朋友或同事。

## 核心特性

### 🔐 端到端加密
- 使用 RSA-2048 位密钥加密
- 只有接收方可以解密
- 服务器无法查看明文密码

### 👥 联系人管理
- 保存常用联系人的公钥
- 快速选择分享对象
- 添加备注方便识别

### ⏰ 过期控制
- 1小时 / 1天 / 7天 / 永不过期
- 过期后自动失效
- 可随时撤销分享

### 🔑 权限控制
- 可查看：接收方可以查看密码
- 可保存：接收方可以保存到自己的密码库
- 可撤销：发送方可以随时撤销

## 使用方法

### 第一步：添加联系人

#### 方式1：扫描二维码

1. 打开 SafeVault
2. 点击「我的联系人」
3. 点击「+」添加联系人
4. 选择「扫描二维码」
5. 扫描对方的「身份码」
6. 输入备注（如"妈妈"、"同事小王"）
7. 点击「保存」

#### 方式2：手动输入

1. 在添加联系人界面
2. 点击「手动输入」
3. 粘贴对方的身份码内容
4. 点击「确认」

### 第二步：分享密码

1. 打开密码列表
2. 长按要分享的密码
3. 选择「分享」
4. 从联系人列表选择接收方
5. 选择有效期：
   - 1小时（适合临时分享）
   - 1天（适合短期分享）
   - 7天（适合一般分享）
   - 永不（适合常用联系人）
6. 验证身份（指纹/密码）
7. 显示二维码
8. 让接收方扫描二维码

### 第三步：接收分享

1. 扫描发送方的二维码
2. SafeVault 自动打开
3. 查看分享的密码信息
4. 点击「保存到我的密码库」
5. 完成！

## 我的功能码在哪里？

### 查看我的身份码

1. 打开 SafeVault
2. 点击「我的联系人」
3. 点击「我的身份码」
4. 显示包含你公钥的二维码
5. 其他人扫描此码即可添加你为联系人

### 分享身份码

1. 在「我的身份码」页面
2. 点击「分享身份码」
3. 选择分享方式（微信、邮件等）

## 常见问题

### Q: 分享安全吗？
**A:** 是的！SafeVault 使用端到端加密，你的密码在传输过程中始终保持加密状态，只有接收方的私钥可以解密。

### Q: 服务器能看到我的密码吗？
**A:** 不能。SafeVault 采用零知识架构，服务器永远无法访问明文密码。

### Q: 如果我换了手机，旧分享还有效吗？
**A:** 有效！因为密钥对是从主密码确定性派生的，只要使用相同的主密码，所有设备都会生成相同的密钥对。

### Q: 可以撤销分享吗？
**A:** 可以！在「分享历史」中可以查看并撤销之前创建的分享。

### Q: 接收方必须安装 SafeVault 吗？
**A:** 是的，接收方需要安装 SafeVault 才能扫描和解密分享。

### Q: 分享的密码有大小限制吗？
**A:** 有。由于 RSA 加密的限制，分享的数据不能超过约 245 字节。对于包含大量信息的密码条目，建议使用离线文件分享。

### Q: 离线文件分享如何使用？
**A:**
1. 选择联系人后点击「生成离线文件」
2. 文件会保存为 .svsf 格式
3. 通过U盘、文件传输应用等方式发送
4. 接收方用 SafeVault 打开文件即可

### Q: 联系人的公钥存在哪里？
**A:** 联系人的公钥加密存储在你的本地设备上，不会上传到服务器。

## 安全提示

### ⚠️ 重要提醒

1. **只分享给可信任的人**
   - 端到端加密无法防止接收方恶意泄露密码

2. **谨慎使用永久分享**
   - 建议设置合理的过期时间
   - 定期检查和清理不再需要的分享

3. **保护你的主密码**
   - 主密码是所有密钥的根源
   - 不要与他人分享主密码

4. **验证联系人身份**
   - 在添加联系人时，确认对方身份
   - 最好当面或通过可信渠道交换身份码

5. **定期检查分享记录**
   - 在「分享历史」中查看所有分享
   - 及时撤销不需要的分享

## 技术细节（供高级用户参考）

### 加密算法
- **密钥派生**: PBKDF2-HMAC-SHA256 (100,000迭代)
- **密钥类型**: RSA-2048
- **加密模式**: RSA-OAEP with SHA-256
- **签名算法**: SHA256withRSA

### 数据格式
- **身份码**: `safevault://identity/{base64(json)}`
- **分享码**: `safevault://share/{base64(encrypted_data)}`

### 隐私保护
- 位置信息不会被收集
- 联系人关系存储在本地
- 分享记录加密存储
