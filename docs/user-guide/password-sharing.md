# SafeVault 密码分享功能用户指南

## 功能概述

SafeVault 支持**端到端加密**的密码分享功能，仅支持与好友分享密码，确保分享的安全性和可控性。

## 核心特性

### 🔐 端到端加密
- 使用 RSA-2048 位密钥加密
- 只有接收方可以解密
- 服务器无法查看明文密码

### 👥 好友分享
- 仅支持与已添加的好友分享
- 先添加好友，再分享密码
- 确保分享对象可信赖

### ⏰ 过期控制
- 1小时 / 1天 / 7天 / 永不过期
- 过期后自动失效
- 可随时撤销分享

### 🔑 权限控制
- 可查看：接收方可以查看密码
- 可保存：接收方可以保存到自己的密码库
- 可撤销：发送方可以随时撤销

## 使用方法

### 第一步：添加好友

#### 方式1：搜索用户名

1. 打开 SafeVault
2. 点击「好友」
3. 点击「+」添加好友
4. 输入对方的用户名
5. 点击「搜索」
6. 选择「发送好友请求」

#### 方式2：扫描二维码

1. 点击「扫描二维码」
2. 扫描对方的「身份码」
3. 系统自动发送好友请求

#### 方式3：通过分享码

1. 让对方分享他们的「身份码」
2. 在添加好友界面选择「输入分享码」
3. 粘贴对方的分享码
4. 点击「添加」

### 第二步：等待好友请求被接受

好友请求发送后，对方会收到通知：
1. 对方在「好友」中看到你的请求
2. 对方点击「接受」
3. 双方成为好友，可以互相分享密码

### 第三步：分享密码

1. 打开密码列表
2. 点击要分享的密码
3. 点击「分享」按钮
4. 从好友列表中选择接收方（只能选择已添加的好友）
5. 设置权限：
   - 是否允许查看
   - 是否允许保存
   - 是否可撤销
6. 选择有效期：
   - 1小时（适合临时分享）
   - 1天（适合短期分享）
   - 7天（适合一般分享）
   - 永不（适合常用好友）
7. 验证身份（指纹/密码）
8. 分享成功！对方会收到通知

### 第四步：接收分享

1. 当好友分享密码给你时，你会收到通知
2. 打开通知或在「分享」中查看
3. 查看分享的密码信息
4. 点击「保存到我的密码库」
5. 完成！

## 查看分享历史

### 我创建的分享

1. 点击「分享」
2. 点击「已发送」标签
3. 查看所有发送给好友的分享
4. 可以撤销可撤销的分享

### 我接收的分享

1. 点击「分享」
2. 点击「已接收」标签
3. 查看所有好友分享给你的密码
4. 可以将分享保存到密码库

## 我的身份码在哪里？

### 查看我的身份码

1. 打开 SafeVault
2. 点击「我的」
3. 点击「我的身份码」
4. 显示包含你公钥的二维码
5. 其他人扫描此码即可添加你为好友

### 分享身份码

1. 在「我的身份码」页面
2. 点击「分享」
3. 选择分享方式（微信、QQ等）

## 常见问题

### Q: 分享安全吗？
**A:** 是的！SafeVault 使用端到端加密，你的密码在传输过程中始终保持加密状态，只有接收方的私钥可以解密。

### Q: 为什么必须先添加好友？
**A:** 为了确保分享的安全性，我们要求只能与已添加的好友分享密码。这样可以：
- 防止陌生人随意请求分享
- 确保你认识分享对象
- 提高分享的可控性

### Q: 服务器能看到我的密码吗？
**A:** 不能。SafeVault 采用零知识架构，服务器永远无法访问明文密码。

### Q: 可以撤销分享吗？
**A:** 可以。在「分享」→「已发送」中找到要撤销的分享，点击「撤销」即可。

### Q: 接收方必须安装 SafeVault 吗？
**A:** 是的，接收方需要安装 SafeVault 并成为你的好友才能接收分享。

### Q: 好友请求会过期吗？
**A:** 好友请求不会自动过期，但建议定期清理未处理的请求。

### Q: 可以删除好友吗？
**A:** 可以。在「好友」列表中，点击好友旁边的「更多」→「删除好友」。删除后不能再与该好友分享密码。

### Q: 删除好友后，之前的分享还有效吗？
**A:** 删除好友关系不会影响已创建的分享。已发送的分享仍然有效，直到过期或被撤销。

### Q: 可以同时分享给多个好友吗？
**A:** 目前一次只能分享给一个好友。如需分享给多人，请分别创建分享。

### Q: 好友关系存储在哪里？
**A:** 好友关系存储在云端服务器，确保在不同设备间同步。

## 安全提示

### ⚠️ 重要提醒

1. **只与可信赖的人添加好友**
   - 端到端加密无法防止接收方恶意泄露密码
   - 添加好友前确认对方身份

2. **谨慎使用永久分享**
   - 建议设置合理的过期时间
   - 定期检查和清理不再需要的分享

3. **保护你的主密码**
   - 主密码是所有密钥的根源
   - 不要与他人分享主密码

4. **定期检查好友列表**
   - 删除不认识或不再联系的好友
   - 撤销不再需要的分享

5. **验证好友身份**
   - 在添加好友时，确认对方身份
   - 最好当面或通过可信渠道交换身份码

## 技术细节（供高级用户参考）

### 加密算法
- **密钥派生**: PBKDF2-HMAC-SHA256 (100,000迭代)
- **密钥类型**: RSA-2048
- **加密模式**: RSA-OAEP with SHA-256
- **签名算法**: SHA256withRSA

### 数据格式
- **身份码**: `safevault://identity/{base64(json)}`
- **分享数据**: 加密存储在服务器，只有接收方私钥可解密

### 隐私保护
- 密码数据始终加密存储
- 好友关系云端存储（用于同步）
- 分享记录加密存储

### 网络通信
- 使用 HTTPS 加密传输
- JWT 令牌认证
- WebSocket 实时通知
