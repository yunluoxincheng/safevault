# fix-sharing-bugs Proposal

## Summary

修复 SafeVault 联系人和分享功能中的 7 个关键 bug，确保用户体验流畅：
1. 注册后邮箱未保存导致身份码无法生成
2. 删除好友功能无法正常工作
3. 搜索用户时未过滤已添加的好友
4. 联系人分享时无法打开联系人选择界面
5. 缺少扫描分享二维码的入口
6. 蓝牙设备搜索显示问题
7. 完全移除 NFC 功能（Android 10+ 不支持点对点传输）

## Motivation

经过测试发现的这些 bug 严重影响了用户体验：
- **问题1** 导致用户注册后无法使用身份码功能
- **问题2** 导致用户无法删除不需要的好友
- **问题3** 导致用户可能重复添加好友，产生混淆
- **问题4** 导致联系人分享功能完全不可用
- **问题5** 导致用户无法通过二维码快速接收分享
- **问题6** 导致蓝牙分享体验不佳
- **问题7** NFC 功能在 Android 10+ 已无法使用点对点传输，保留只会造成混淆

## Scope

### In Scope
- 修复注册后邮箱信息未保存的问题
- 修复删除好友功能，同时删除云端和本地数据
- 添加搜索用户时过滤已添加好友的逻辑
- 修复联系人选择界面的启动问题
- 添加扫描分享二维码的入口和功能
- 改进蓝牙设备搜索显示，标记已配对和新设备
- **完全移除 NFC 相关代码和 UI 入口**

### Out of Scope
- 不涉及新的分享方式实现
- 不修改现有的分享加密逻辑
- 不涉及后端 API 修改（除调用现有 API 外）

## Dependencies

- 依赖于 `contact-sharing` 规范中的好友关系管理
- 依赖于 `backend-integration` 规范中的 API 集成

## Risks

- **邮箱保存修复**: 需要确保不影响现有登录流程
- **删除好友修复**: 需要同时处理云端和本地数据，避免数据不一致
- **蓝牙搜索改进**: 需要正确处理权限和设备发现
- **NFC 移除**: 需要确保不影响其他功能

## Timeline Estimate

- 问题 1-4: 核心功能修复，预计 2-3 小时
- 问题 5: 扫码入口添加，预计 1 小时
- 问题 6: 蓝牙搜索改进，预计 1-2 小时
- 问题 7: NFC 功能移除，预计 1 小时

**总计**: 约 5-7 小时

## Success Criteria

1. ✅ 用户注册后能在"我的身份码"页面看到二维码
2. ✅ 用户可以成功删除好友，本地和云端同步删除
3. ✅ 搜索用户时，已添加的好友不显示在搜索结果中
4. ✅ 点击"选择联系人"能正常打开联系人列表
5. ✅ 主界面有"扫一扫"入口，可以扫描分享二维码
6. ✅ 蓝牙设备搜索时，已配对和新设备有明确标识
7. ✅ 所有 NFC 相关代码和 UI 入口已移除

## Approaches Considered

### 问题1: 邮箱未保存
**选项1 (已选)**: 在 `RegisterActivity` 注册成功后调用 `TokenManager.saveLastLoginEmail()`
**选项2**: 修改 `MyIdentityActivity` 从其他来源获取邮箱（不可行，邮箱只在注册时获得）

### 问题2: 删除好友
**选项1 (已选)**: 在 `ContactListActivity` 中先调用云端 API 删除，成功后再删除本地数据
**选项2**: 只删除本地数据（不可行，会导致云端和本地数据不一致）

### 问题3: 搜索过滤
**选项1 (已选)**: 在 `ContactSearchActivity` 中获取本地联系人列表，过滤已存在的用户
**选项2**: 依赖后端 API 返回时过滤（不可行，后端可能不返回好友状态）

### 问题4: 联系人选择
**选项1 (已选)**: 修复 `ShareActivity` 中的 `startActivityForResult` 调用
**选项2**: 使用 ActivityResult API 替代（更好的实践，但需要更多改动）

### 问题5: 扫码入口
**选项1 (已选)**: 在 `MainActivity` 添加 FAB 按钮"扫一扫"
**选项2**: 在设置中添加入口（不够直观）

### 问题6: 蓝牙搜索
**选项1 (已选)**: 在 `BluetoothDeviceListDialog` 中区分已配对和新发现的设备
**选项2**: 分两个列表显示（UI 更复杂）

### 问题7: NFC 移除
**选项1 (已选)**: 完全移除 NFC 相关代码和 UI 入口
**选项2**: 保留但添加说明（不可行，功能不可用会造成混淆）
