# 简化联系人分享系统

## 摘要
简化 SafeVault 的密码分享功能，移除直接链接分享和附近用户分享功能，仅保留基于好友关系的联系人分享。此变更将删除 `password_shares` 和 `online_users` 数据库表，创建新的 `contact_shares` 表，并移除前端和后端中相关的代码。

## 背景
当前分享系统支持三种分享方式：
1. **直接链接分享 (DIRECT)**：生成链接任何人可访问
2. **用户对用户分享 (USER_TO_USER)**：指定用户分享，但不限制好友关系
3. **附近用户分享 (NEARBY)**：基于位置的附近用户发现

**问题**：
- 好友系统与分享系统解耦，任何人都可以分享，削弱了好友系统的价值
- 附近用户功能需要位置权限，增加隐私风险
- 直接链接分享存在安全风险
- 代码复杂度高，维护成本大

## 目标
1. **简化分享模型**：仅支持好友间的密码分享
2. **提高安全性**：确保只有好友才能分享和接收密码
3. **降低复杂度**：删除不需要的表和代码
4. **保持功能完整性**：联系人分享的核心功能（权限、撤销、过期）保持不变

## 影响范围

### 后端 (safevault-backend)
| 模块 | 变更类型 | 说明 |
|------|----------|------|
| `password_shares` 表 | 删除 | 被 contact_shares 替代 |
| `online_users` 表 | 删除 | 附近用户功能完全移除 |
| `share_audit_logs` 表 | 删除 | 审计日志不再需要 |
| `contact_shares` 表 | 新增 | 新的联系人分享表 |
| `ShareService` | 重构 | 简化为仅支持联系人分享 |
| `DiscoveryService` | 删除 | 附近用户发现服务 |
| `DiscoveryController` | 删除 | 附近用户 API |
| `ShareController` | 修改 | API 接口调整 |
| 数据库迁移 | 新增 | Flyway 迁移脚本 |

### 前端 (SafeVault Android)
| 模块 | 变更类型 | 说明 |
|------|----------|------|
| `ShareActivity` | 修改 | 移除分享方式选择，仅联系人分享 |
| `NearbyUsersActivity` | 删除 | 附近用户界面 |
| `ShareViewModel` | 简化 | 移除 DIRECT 和 NEARBY 类型 |
| `ShareType` 枚举 | 删除 | 仅保留 USER_TO_USER 逻辑 |
| DTO 类 | 简化 | 移除不需要的字段 |

### 数据库迁移
| 脚本 | 说明 |
|------|------|
| `VX__drop_password_shares.sql` | 删除旧表 |
| `VX__drop_online_users.sql` | 删除附近用户表 |
| `VX__drop_share_audit_logs.sql` | 删除审计日志表 |
| `VX__create_contact_shares.sql` | 创建新表 |

## 不在范围内
- 好友系统 (friendships, friend_requests) 保持不变
- 密码库 (user_vaults) 保持不变
- 用户认证系统保持不变
- WebSocket 通知系统保持不变（通知逻辑简化）

## 风险评估
| 风险 | 级别 | 缓解措施 |
|------|------|----------|
| 现有分享数据丢失 | 高 | 提供数据迁移脚本 |
| API 兼容性问题 | 中 | 版本号升级，通知前端 |
| 前端功能缺失 | 中 | 逐步移除，确保新功能完整 |

## 验收标准
1. 后端 API 仅支持联系人分享
2. 前端分享界面仅显示联系人选择
3. 附近用户功能完全移除
4. 数据库表结构正确更新
5. 单元测试和集成测试通过
6. 无遗留的死代码

## 依赖
- 好友系统功能完整
- 前端联系人列表功能正常
